🚀 Cómo publicar tu CRM en internet

Guía paso a paso para subir CRM Alma Orders a GitHub, Supabase y Vercel

📁
Estructura de archivos que tenés que tener
Todos estos archivos en UNA sola carpeta llamada crm-alma-orders
📁 crm-alma-orders/
  📄 README.md
  📄 .gitignore
  📄 vercel.json
  📄 supabase-schema.sql
  📁 public/
    📄 index.html ← tu CRM (el archivo grande)
💡 Descargá todos estos archivos desde Claude y ponelos en una carpeta nueva en tu computadora
🐙
PASO 1 — Subir a GitHub github.com
GitHub guarda tu código en la nube. Vercel lo lee desde acá.
1
Creá tu cuenta
Entrá a github.com → clic en Sign up → ponés email + contraseña + nombre de usuario → verificás el email
2
Creá un repositorio nuevo
Dentro de GitHub clic en el botón verde New (o el + arriba a la derecha → New repository)
3
Configurá el repositorio
• Nombre: crm-alma-orders
• Descripción: CRM Alma Orders
• Marcá Public (para que Vercel lo pueda leer gratis)
• NO marques "Add a README" (ya tenés uno)
• Clic en Create repository
4
Instalá GitHub Desktop (la forma fácil sin código)
Bajá desktop.github.com → instalalo → iniciá sesión con tu cuenta de GitHub
💡 GitHub Desktop te permite subir archivos con clics, sin escribir comandos
5
Cloná tu repositorio vacío
En GitHub Desktop → File → Clone repository → buscás crm-alma-orders → elegís dónde guardarlo en tu PC → Clone
6
Copiá tus archivos a la carpeta clonada
Abrí la carpeta que creó GitHub Desktop en tu computadora (hace clic en Show in Explorer / Finder) y copiá ahí todos los archivos: README.md, .gitignore, vercel.json, supabase-schema.sql, y la carpeta public/ con index.html adentro.
7
Subí los archivos a GitHub
Volvé a GitHub Desktop → vas a ver todos los archivos listados en el panel izquierdo → abajo a la izquierda escribís un mensaje como Primer commit - CRM Alma Orders → clic en Commit to main → después clic en Push origin
✅ ¡Listo! Tus archivos ya están en GitHub. Podés verlos en github.com/TU-USUARIO/crm-alma-orders
🗄️
PASO 2 — Configurar Supabase supabase.com
Supabase crea tu base de datos donde se guardan todos los registros del CRM
1
Creá tu cuenta
Entrá a supabase.com → Start your project → Continue with GitHub → autorizás → ya entraste
2
Creá un nuevo proyecto
Clic en New project → ponés:
• Nombre: crm-alma-orders
• Contraseña de base de datos: anotala en algún lado (la necesitás después)
• Región: South America (São Paulo) — la más cercana a Argentina
→ Clic en Create new project → esperás 2 minutos que se cree
⚠️ Guardá la contraseña en un bloc de notas. Si la perdés hay que resetearla.
3
Creá las tablas con el SQL
Dentro de tu proyecto Supabase, en el menú izquierdo hacé clic en SQL Editor → clic en New query → copiá TODO el contenido del archivo supabase-schema.sql → pegalo en el editor → clic en Run (el botón verde)
✅ Si dice "Success. No rows returned" está todo bien. Las tablas se crearon.
4
Copiá tu URL y clave pública (anótalas)
En el menú izquierdo → Project Settings → API

Ahí vas a ver:
• Project URL: algo como https://xyzabc.supabase.co
• anon public key: una cadena larguísima de letras

Copiá ambas y guárdalas — las vas a necesitar para conectar el CRM
5
Verificá que las tablas existen
En el menú izquierdo → Table Editor → deberías ver: profesionales, tiendas, comercios, riders, embajadores, prospectos, actividad
✅ Si las ves todas, ¡Supabase está listo!
▲
PASO 3 — Publicar en Vercel vercel.com
Vercel toma tu código de GitHub y lo pone en internet con una URL real
1
Creá tu cuenta
Entrá a vercel.com → Sign Up → Continue with GitHub → autorizás → ya entraste
2
Importá tu proyecto de GitHub
En el panel de Vercel → clic en Add New... → Project → va a aparecer una lista de tus repositorios de GitHub → buscás crm-alma-orders → clic en Import
3
Configurá el deploy
• Framework Preset: Other
• Root Directory: dejalo vacío
• Build Command: dejalo vacío
• Output Directory: public
→ Clic en Deploy
4
Esperá 1-2 minutos
Vercel va a mostrar una animación mientras procesa. Cuando termine vas a ver una pantalla con ✅ y una URL como:
crm-alma-orders.vercel.app
✅ ¡Tu CRM ya está en internet! Cualquiera con esa URL puede abrirlo.
5
Probá que funciona
Abrí la URL en tu celular o en otra computadora → vas a ver la pantalla de login → usuario: admin → contraseña: admin → entrás al CRM
🔄
Cómo actualizar el CRM en el futuro
Cada vez que cambiás algo y lo querés ver en internet
1
Modificás el archivo index.html en tu computadora
Hacés los cambios que necesites en el archivo local
2
Subís el cambio con GitHub Desktop
Abrís GitHub Desktop → ves los cambios listados → escribís un mensaje describiendo qué cambiaste → Commit to main → Push origin
3
Vercel actualiza solo automáticamente
Vercel detecta el cambio en GitHub y en 1-2 minutos tu URL pública ya muestra la versión nueva. No tenés que hacer nada más.
✅ Así funciona el deploy continuo: GitHub → Vercel → internet, automático.
🔐
Cómo ingresar al CRM
Credenciales de acceso al sistema
🌐
URL de acceso
https://crm-alma-orders.vercel.app (o la URL que te dio Vercel)
👤
Usuario y contraseña (demo)
Actualmente el login acepta cualquier usuario y contraseña — es un demo. Para agregar login real con Supabase Auth se puede implementar en una segunda fase.
⚠️ Mientras sea solo un HTML estático, cualquiera que tenga la URL puede entrar. Si necesitás privacidad real, avisame y agregamos autenticación con Supabase.
📱
Funciona en celular
Podés abrir la URL desde cualquier celular o tablet. El CRM es responsive — se adapta a cualquier pantalla.
